Полезные инструменты и советы для решения учебных задач с курса:
Инструменты:
1) Осторожное использование библиотеки Lombok поможет писать более краткий и читаемый код.
2) Осмысленное объявление классов рекордами поможет писать более краткий и читаемый код.
3) Иногда данный импорт может помочь: import java.util.concurrent.atomic.AtomicInteger;.
4) Иногда данный импорт может помочь: import java.math....;.
5) Валидацию можно реализовывать разными способами:
   выносить в методы, вешать аннотации или использовать библиотеку 'org.apache.commons'.
6) Перед работой с массивами и коллекциями нужно просматривать через IDEA все их методы,
   так как тебе нужно освежить память перед работой.
   а) Основные методы интерфейса List:
   add; add(int index, E element); get; set; remove; remove(Object o); size; contains; indexOf; isEmpty.
   Цикл for-each; Интерфейс Iterator; subList; sort.
   б) Основные методы интерфейса Set:
   add; remove; contains; size; isEmpty; clear; iterator; addAll; retainAll; removeAll.
   equals; hashCode; Set.of.
   в) Основные методы интерфейса Map:
   put; get; remove; containsKey; containsValue; size; isEmpty; clear; putAll; keySet; values; entrySet
   Цикл for + entrySet; equals; hashCode.
7) Умение использовать copyOf значительно упрощает рутинный и сложный код.
8) Знание метода computeIfAbsent значительно упрощает рутинный и сложный код.
9) Знание метода getOrDefault значительно упрощает рутинный и сложный код.
10) Умение использовать var значительно упрощает рутинный и сложный код.
11) Чёткие контракты коллекций: «снимок» (snapshot) vs «живой вид» (live view). 
    Наружу — только снимки/неизменяемые коллекции.
12) Инъекция зависящих от среды вещей (время/случайность/IO): тесты становятся проще.

Советы:
1) Цепочка действий до того как начнёшь писать код:
   Прочитай задачу несколько раз, тебе нужно понять из
   неё все термины и требования для дальнейшей работы;
   Составляй краткий список всех требований из задачи,
   тебе нужно иметь перед глазами всю выжимку;
   Дополняй список своими мыслями и идеями,
   тебе нужно закрыть логические пробелы;
   Набросай схему архитектуры*, когда почувствуешь
   что у тебя накопилось достаточно информации;
   *Схема архитектуры: поля методы класс, поля методы класс,
   поля методы класс, затем разбрасываешь всё по пакетам;
   Сделай паузу и подумай о возможных подводных камнях,
   сделай заметки и напиши возможные альтернативные решения;
   Проектируй повторную осмысленную схему архитектуры;
   +Выделяй достаточно времени на качественное продумывание
   названий для пакетов, классов и полей с методами.
2) Если чувствуешь, что архитектура сложна, то начни
   её реализацию с упрощённого вида, например временно
   опусти валидацию, нормализацию и попробуй "большой" метод.
